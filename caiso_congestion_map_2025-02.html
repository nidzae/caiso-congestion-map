What is this?

An interactive screening map of CAISO pricing nodes designed to identify where battery storage could relieve transmission congestion. Each node is encoded by four channels (hue, saturation, size, marker style) derived from the marginal congestion component (MCC) of locational price, not the total price. This is a research instrument; deliverable relief requires a network-model run beyond scope.

The encoding (four channels)

1. Hue → archetype (flow direction)

  • ⬤ WHITE — no local congestion; price swing is purely energy-driven
  • ⬤ BLUE — import pocket (evening MCC high; binding constraint feeds load)
  • ⬤ RED — export pocket (midday MCC negative; trapped supply, e.g. solar oversupply)
  • ⬤ PURPLE — bidirectional / double-duty (both import and export congestion). Most interesting for siting.

2. Saturation → round-trip MCC spread (intensity)

Within each archetype, nodes are sorted into spread quintiles (q0–q4) — higher quintile = more saturated color. The quintile-based mapping prevents the long tail of extreme spreads (Kern County, etc.) from washing out the middle of the distribution. 3. Marker size → controlling-constraint rent

Radius ∝ √(size_node) where size_node = rating × Σ|μ| for the constraint that controls the node (its "k*"). Bigger circle = larger congestion prize on the wire this node sits on.

4. Marker style → bankability (concentration)

filled circle — persistent congestion value (most days/hours)
hollow ring — spike-driven value (rare outage hours dominate). Less bankable.

Threshold: hollow if top 1% of 5-min intervals contributes more than 50% of total |MCC| value.

Battery duration (the 2 h / 4 h / 8 h buttons)

The picker at the top-left of the map switches the battery duration D used to compute the round-trip spread (channel #2, color saturation). It changes how the map colors each node — same coordinates, same archetype, but a different intensity reflecting how much arbitrage a battery of that duration would capture at that node.

Why duration matters: a battery's daily arbitrage value depends on how many contiguous high/low hours it can ride. Different node behaviors favor different durations:

  • Spike-driven nodes have a handful of very-high-price hours per day — a short (2 h) battery can capture most of the value. Adding more duration unlocks little extra revenue.
  • Persistent-spread nodes have wider price differentials sustained over many hours — a long (8 h) battery unlocks meaningfully more revenue than a 2 h would. Path-26-fed LA Basin nodes like Alamitos sit here: the import constraint binds for most of the 4–10 pm window, so an 8 h battery has more discharge hours to exploit.

How to see this in the bar chart (📊 Ranked tab): each top-100 bar is a stacked composite showing where the value comes from:

  • Innermost (faintest) segment = the D=2 h spread — the spike value a short battery captures.
  • Middle segment = the incremental value from doubling 2 h → 4 h.
  • Outermost (most saturated) segment = the incremental value from doubling 4 h → 8 h. Total bar length = D=8 h spread.

Quick reads:

  • If a bar is dominated by its innermost segment, the node is spike-driven — there's little benefit to building anything beyond a 2 h system there.
  • If the outer two segments are visibly thick, the node is duration-sensitive — longer storage genuinely unlocks more value (often points to a constraint that binds for many contiguous hours).
  • Hover any segment to see its specific contribution and the full 2 h / 4 h / 8 h sweep.

Hover field reference

  • archetype — flow-direction classification (WHITE / BLUE / RED / PURPLE)
  • spread — median daily round-trip arbitrage value of MCC, $/MWh. The bigger the spread, the more $$$ a battery sitting there could capture per day.
  • (q0–q4) — spread quintile within this archetype (saturation level)
  • size — rating × Σ|μ| over the month, in $. Total congestion-rent prize on the wire this node controls.
  • controlling line — physical transmission element with largest β·μ contribution from the regression. OASIS nomogram or intertie ID.
  • k* rating — MW capacity of that line. Sources: WECC named path catalog (gold), explicit intertie MVA, or voltage-class proxy (69→150, 115→250, 230→700, 500→2500).
  • conc — share of total |MCC| value contained in the top 1% of 5-min intervals. High conc → spike-driven (hollow).
  • marker — filled / hollow per conc.
  • EIA plant — fuzzy-matched generator name from EIA-860 used for coordinates. Some long-tail matches are approximate.
  • lat,lon — matched plant coordinates.

Core equations (brief §3)

Price decomposition (CAISO RTM 5-min)

LMP(t) = MEC(t) + MCCnode(t) + MLCnode(t) + GHG(t)
t
time, in 5-min intervals (CAISO real-time market).
LMP(t)
locational marginal price at the node, in $/MWh — the total price a generator gets paid (or a load pays) for energy at this location.
MEC(t)
marginal energy component — system-wide reference price; identical at every node, so it carries no location information.
MCCnode(t)
marginal congestion component — the only piece that varies by location, set by which transmission constraints are binding. This is the signal we use throughout.
MLCnode(t)
marginal loss component — covers transmission line losses; small.
GHG(t)
greenhouse-gas cost adder (cap-and-trade pass-through). Only nonzero hours when GHG-emitting units are on the margin.

Daily spread (per node)

discharge(d) = sum of the D highest hourly MCC values on day d charge(d) = sum of the D lowest hourly MCC values on day d daily_spread(d) = discharge(d) − (1/η) × charge(d) spread_node = median over days of daily_spread(d)
D
battery duration in hours — how long the battery can keep discharging at rated power. Default D = 4 (a 4-hour battery).
η
round-trip efficiency — fraction of energy that survives one charge + discharge cycle. Default η = 0.85 (15% losses). The factor (1/η) means you have to charge slightly more MWh than you'll later discharge.
MCC (hourly)
marginal congestion component, averaged from 5-min to hourly. In $/MWh.
d
calendar day index.

MCC stays unclipped — negative MCC during charge hours makes charge negative, so −(1/η) × charge turns into a positive contribution: the battery is effectively paid to charge and that's part of the prize.

Archetype classification

MCC_mid = mean MCC over 10:00–15:00 local time (midday window) MCC_eve = mean MCC over 17:00–21:00 local time (evening window) import_signal = MCC_eve export_signal = −MCC_mid if both signals < ε: WHITE (no local congestion) if import ≥ ε, export < ε: BLUE (import pocket) if export ≥ ε, import < ε: RED (export pocket) if both signals ≥ ε: PURPLE (bidirectional)
ε
flatness threshold in $/MWh — signals below ε are treated as "no congestion in that direction". Default ε = $3/MWh.
import_signal
positive when the node sits behind a constraint that binds when load is high (evening peak).
export_signal
positive when midday MCC is negative — happens when local generation can't get out because the wire to the rest of the grid is full (classic solar oversupply).

Attribution & sizing

MCCnode(t) ≈ Σk βnode,k · μk(t) (Ridge regression) k* = argmaxk meant |βnode,k · μk(t)| (the "controlling" physical line) size_node = rating[k*] × Σt |μk*(t)| (the dollar prize on that line)
μk(t)
shadow price of constraint k at time t, in $/MWh — CAISO publishes this. When a constraint binds, μ is the $/MWh the market would save if you could relax it by 1 MW. μ = 0 when the constraint isn't binding.
k
index over binding transmission constraints (in our August 2025 panel: 287 distinct nomogram + intertie shadow-price series).
βnode,k
regression coefficient — how much this node's MCC moves per $1 of shadow price on constraint k. Fitted by Ridge regression. Conceptually an empirical shift factor (PTDF); not exact PTDFs.
k*
the constraint whose contribution |β · μ|, averaged over time, is largest for this node — i.e. the wire that controls this node's congestion. Aggregated to the physical-line level (scenarios combined).
rating[k*]
nameplate MW capacity of the controlling line, from the WECC Path Rating Catalog or a voltage-class proxy.
Σt |μk*(t)|
sum, over every 5-min interval in the month, of the absolute value of the controlling line's shadow price. Bigger means the line binds more often or harder — equivalently, total congestion-price "volume" on the line. Multiplied by the rating, this gives the total monthly congestion rent in dollars.
Σk
sum over all constraints k in the panel.
argmaxk
"the value of k that makes the following expression largest" — picks the dominant constraint.

Bankability (concentration)

conc = (sum of |MCC| in the top 1% of 5-min intervals) / (total sum of |MCC| over all intervals) hollow marker if conc > 0.5
|MCC|
absolute value of the marginal congestion component (we use |MCC| because both positive — import — and negative — export — MCC are valuable to a battery).
top 1%
over a 31-day month at 5-min resolution there are 8,928 intervals, so the top 1% is the ~89 most extreme intervals.
conc threshold (0.5)
if more than half of all congestion value comes from those ~89 spike intervals, the node is "spike-driven" and rendered as a hollow ring. Otherwise it's persistent (filled circle).

Data sources

  • CAISO OASIS (accessed via the gridstatus Python library): 5-min RTM LMP with congestion component, plus nomogram + intertie shadow prices for August 2025.
  • WECC 2026 Path Rating Catalog (Public Version): named transmission-path MW ratings.
  • EIA-860 (2025 Early Release): generating-plant coordinates.

⚠ Important: plant matches are fuzzy

CAISO publishes prices by pricing-node name (e.g. ALAMT1G_7_B1), not coordinates. We attach a lat/lon by fuzzy-matching the abbreviation to the EIA-860 plant catalog:

  • For ~10 well-known LA Basin plants (Alamitos, Harbor, Etiwanda, Huntington Beach, El Segundo, Ormond, Sentinel, …), we use a hand-curated alias map — these placements are reliable.
  • For the rest, we use a subsequence-density matcher with three guards: (1) abbreviation must match the plant name with character density ≥ 0.5, (2) the EIA plant must be in the right Balancing Authority (CISO for SP15/NP15/ZP26; PACE/PACW for their hubs), with state fallback only if no BA match exists, and (3) when 3+ different CAISO abbreviations all match the same plant, all of them are demoted to "unplaced" because at least most of them are wrong.

Even with those guards, a handful of placements are still wrong. CAISO uses 5–8 char mnemonics that don't always map cleanly to EIA's full plant names, and many CAISO pnodes are for substations (Vincent, Lugo, Mira Loma) that don't appear in EIA-860 at all — those are intentionally left unplaced.

Always hover-check the "EIA plant" field before acting on a placement. If the plant name doesn't match what you'd expect for that node, treat the location as suspect. The bar-chart tab is the safer view if you only care about the rankings.

Why some dots stack with different colors

The map dot is a geographic point, but MCC is computed at a specific electrical bus in the network. One physical substation can have multiple distinct buses sharing the same lat/lon, and each bus generally has its own shift factors (PTDFs) to the binding constraints — so their MCCs can differ.

Concrete sources of within-site divergence:

  • Different voltage levels (500 kV / 230 kV / 115 kV yards) at one substation. Each yard is a distinct bus and only "sees" the constraints electrically coupled to it.
  • A transformer between two buses can itself bind — when it does, the high-side and low-side prices diverge by the transformer's shadow price.
  • Split-bus substations with sectionalizing tie breakers; under contingencies the two sections can be electrically separated.
  • Generator-tap buses just upstream of the main station bus, through unit transformers — same address, different injection point in the model.

So when you see a small cluster of dots in the same spot with different colors (e.g., one BLUE + one WHITE at Caribou hydro), it's usually the network model showing through — not a data bug. The pnode IDs and bus suffixes (_B1 vs _B2) usually indicate which bus is which.

Other limitations

  • This is a screening tool. Map says "MCC pattern here suggests congestion." It does not confirm a battery sited there can actually relieve the constraint — that needs a network-model run with PTDFs.
  • Empirical β are not true shift factors; they're regression-fit proxies.
  • Aggregation nodes (DLAPs, sub-LAPs, trading hubs) excluded from this run.
  • One season only (August 2025); seasonal variation not shown.
